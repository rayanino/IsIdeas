# PR Review Checklist for Codex Output

Use this checklist before merging a Codex-generated PR.

## Task Packet Compliance
- did Codex actually follow the assigned objective?
- did it stay inside the task boundary?
- did it avoid inventing standards or strategy?

## Workshop Rule Compliance
- does the change respect `QUALITY_BAR.md`?
- does it respect `ANTI_CHAOS_RULES.md`?
- does it respect `CONSISTENCY_AND_SOURCE_OF_TRUTH.md`?
- does it respect the workspace contract?

## Meaning Preservation
- did the PR preserve idea meaning where only structural work was requested?
- did it quietly alter stage, priority, or judgment language?
- did it flatten dossier nuance into summary language?

## Consistency
- are registry, workspace README, dossier, focus, and priority artifacts still aligned?
- did the PR introduce new drift while fixing old drift?

## Scope and Cleanliness
- is the PR appropriately narrow?
- are there extra edits that should not have been included?
- does every changed file serve the task?

## Merge Decision
- merge
- request revision
- reject

## Notes
Record anything that future review should remember.
