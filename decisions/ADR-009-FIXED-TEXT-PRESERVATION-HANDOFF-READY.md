# ADR-009 — Fixed-Text Preservation System is handoff-ready for MVP build

## Status
Accepted

## Decision
Promote `Fixed-Text Preservation System` from `spec_ready` to `handoff_ready`.

## Why

The idea now has:

- a mature dossier
- an implementation-grade spec
- explicit correctness constraints
- a provisional Qur'an reference contract
- and a concrete implementation packet

That is enough to hand the MVP to a builder without asking them to invent the core behavior.

## Consequence

- `Fixed-Text Preservation System` remains the sole frontier idea
- the next move is implementation, not further concept-level narrowing
- future review should focus on whether the build packet preserved the honesty constraints
