# ideas/

Each idea gets its own workspace directory.

The purpose is to give an idea room to grow without prematurely pretending it is already a final specification.

A typical idea workspace may contain:
- `README.md` — quick status and overview
- `DOSSIER.md` — main working discussion document
- `SPEC.md` — only once promoted
- `RESEARCH.md` — optional research notes when needed

Do not create `SPEC.md` by default.
A spec should be earned.
