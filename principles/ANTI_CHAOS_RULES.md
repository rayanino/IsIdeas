# Anti Chaos Rules

This repository will fail if it becomes organized on the surface but noisy in practice.

These rules exist to prevent that.

## 1. Sparks stay out of the serious registry
Fresh thoughts belong in `capture/`, not in `catalog/IDEA_REGISTRY.md`.
The serious registry is for ideas that already survived an initial judgment.

## 2. One workspace per serious idea
A serious idea gets one home under `ideas/<idea-slug>/`.
Do not scatter its state across random files.

## 3. One serious session should usually have one target
Do not mix many ideas in one deep-work session unless the session is explicitly about repo architecture or cross-cutting comparison.

## 4. Do not promote because of excitement
Promotion happens because the idea became clearer, stronger, and safer, not because it feels inspiring.

## 5. Most ideas should die early
This is healthy.
A repository that never rejects ideas is not selective enough.

## 6. Preserve rejected judgment
If an idea is parked or rejected, record why.
This protects future work from rediscovering the same weak path.

## 7. Limit active fronts
There should usually be:
- one primary idea in active deep focus,
- at most three serious active idea workspaces under heavy attention,
- and no uncontrolled sprawl of simultaneous dossier work.

## 8. Do not create `SPEC.md` casually
A spec is a promotion artifact, not a default file.
If the idea is still fuzzy, it belongs in the dossier.

## 9. Do not let dossiers become graveyards
A dossier must have a current judgment and a next move.
If not, it is decaying.

## 10. Protect the `kr` boundary
Do not let adjacent apps silently absorb substrate responsibilities, and do not stuff workflow logic into `kr` just because it already exists.

## 11. Every serious artifact should reduce ambiguity
If a new file does not make the repo clearer, more disciplined, or more implementable, it is probably noise.

## 12. Prefer depth over intake volume
A repo with ten deeply examined ideas is better than a repo with one hundred loosely captured ones.
