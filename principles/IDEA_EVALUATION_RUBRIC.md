# Idea Evaluation Rubric

Use this rubric before admitting an idea into the serious portfolio.

Score each category from 1 to 5.
An idea does not have to maximize every category, but weak scores should be acknowledged explicitly.

## 1. Strategic Leverage
Does this meaningfully increase the user's long-term scholarly capacity?

- 1: marginal convenience
- 3: useful support tool
- 5: major compounding advantage across years

## 2. Islamic-Study Specificity
Is this genuinely tailored to Islamic study rather than being a generic study tool with Islamic paint?

- 1: generic productivity app
- 3: partially adapted
- 5: deeply rooted in actual study realities

## 3. Cross-Science Applicability
Can it help across multiple sciences or only a narrow niche?

- 1: tiny niche only
- 3: useful within one band of sciences
- 5: broad relevance across study domains

## 4. Correctness Importance
Does this improve correctness, verification, or disciplined reasoning rather than just speed?

- 1: speed only
- 3: some correctness support
- 5: directly reduces risk of error or confusion

## 5. Compounding With `kr`
Does this become more powerful because `kr` exists, and/or strengthen `kr` in return?

- 1: isolated from the core stack
- 3: moderate integration value
- 5: deeply synergistic with the knowledge substrate

## 6. Implementation Clarity
Can this be specified with enough precision that Claude Code can build a credible MVP?

- 1: fuzzy and underdefined
- 3: partly specifiable
- 5: clearly specifiable with strong boundaries

## 7. Human-Review Compatibility
Can the system keep meaningful scholarly judgment under human control where necessary?

- 1: pushes past safe boundaries
- 3: manageable with care
- 5: naturally supports human oversight

## 8. Data Model Integrity
Can the core entities and relationships be represented cleanly?

- 1: model is muddy
- 3: model is workable but messy
- 5: model is crisp and extensible

## 9. MVP Purity
Is there a small first version that proves the real value without bloated scope?

- 1: requires huge system before value appears
- 3: possible but awkward MVP
- 5: crisp MVP with clear validation path

## 10. Risk of Misleading the User
How likely is the tool to create false confidence, shallow understanding, or silent distortion?

- 1: high risk
- 3: moderate risk that needs mitigation
- 5: low risk or well-contained risk

## Decision Guide

### 42–50
Priority candidate. Should likely enter the active roadmap.

### 32–41
Promising, but needs refinement before being treated as core.

### 22–31
Interesting, but not yet strong enough. Rework or deprioritize.

### Below 22
Do not build now.

## Mandatory Written Notes
For each scored idea, also record:
- strongest reason to build it,
- strongest reason not to build it,
- biggest correctness risk,
- main dependency on other systems,
- what the MVP must prove.
