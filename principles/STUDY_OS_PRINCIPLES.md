# Study OS Principles

These principles govern the design of the Islamic-study software portfolio.

## 1. Build a study operating system, not a pile of tools
Each application should either strengthen a shared substrate or remove a major recurring study bottleneck.

## 2. Attack lifelong friction, not one-off inconvenience
Prefer systems that save effort repeatedly across years of study.

## 3. Correctness outranks convenience
A convenient tool that subtly distorts knowledge is worse than no tool.

## 4. Traceability is mandatory
Important outputs must be traceable to sources, excerpts, claims, or explicit user-authored data.

## 5. Uncertainty must be visible
If the system is unsure, it must surface that uncertainty explicitly.

## 6. Human scholarly judgment is not optional
Software should support study, not impersonate a scholar.

## 7. Structure compounds
Topic taxonomies, dependency maps, concept graphs, and review histories become more valuable when shared across systems.

## 8. Retrieval is not enough
A serious study environment must also support sequencing, memorization, comparison, and self-correction.

## 9. The portfolio should reward discipline
Good software should reinforce habits like structured review, careful question capture, and evidence-aware comparison.

## 10. Anti-fragmentation matters
Avoid isolated data silos. Prefer shared primitives and interoperable entities.

## 11. MVPs must be strategically pure
The first version should prove the core leverage, not imitate a full product.

## 12. Failure modes must be designed, not discovered accidentally
Every idea spec should explicitly say how the system can mislead, overreach, or waste time.

## 13. Favor tools that sharpen the student
The best systems do not merely do work for the user. They help the user notice patterns, gaps, dependencies, and weaknesses.

## 14. Preserve the distinction between knowledge, metadata, and inference
A quote is not the same as a claim. A claim is not the same as an interpretation. The system must model these clearly.

## 15. Portfolio coherence beats local optimization
A slightly less flashy app that integrates deeply with the broader study environment is often better than a more impressive isolated one.
