# KR Status

`kr` is a real adjacent project and the intended knowledge substrate, but it is not yet a fully published machine contract for `IsIdeas`.

## Defined Now

- `kr` should own source ingestion, provenance, excerpt linkage, and traceability-critical knowledge representation
- `IsIdeas` apps should own study behavior, memorization workflows, sequencing, and question handling
- the conceptual boundary is documented in `shared/KR_BOUNDARY.md`

## Not Yet Defined Here

- stable IDs and interface contracts exposed from `kr`
- sync/export mechanism between `kr` and `IsIdeas` applications
- build-grade dependency guarantees for idea specs

## Practical Rule

An idea may continue at dossier depth with a conceptual `kr` boundary.

An idea may not pretend to be build-ready if its critical `kr` contract is still undefined.

## Current Exception

`Fixed-Text Preservation System` may proceed into build-facing specification with a provisional Qur'an reference contract.

That exception is allowed only because:

- the MVP is Qur'an-first,
- the reference text is fixed and pin-able,
- the migration debt is explicit,
- and the spec keeps the local contract swappable with future `kr` integration.
