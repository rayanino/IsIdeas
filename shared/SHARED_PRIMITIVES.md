# Shared Primitives

These are candidate cross-portfolio primitives that should stay as consistent as possible across systems.

## Core Identity Primitives
- `science_id`
- `topic_id`
- `source_id`
- `excerpt_id`
- `scholar_id`
- `claim_id`

## Study Process Primitives
- `study_stage_id`
- `review_event_id`
- `memorization_object_id`
- `segment_id`
- `question_item_id`
- `resolution_entry_id`

## Control Primitives
- `uncertainty_state`
- `human_review_state`
- `provenance_record_id`
- `dependency_link_id`

## Rule Of Use

A primitive should be shared only when doing so creates real coherence.
Do not force premature standardization where the underlying concept is still unclear.

## Design Goal

The portfolio should not become a cluster of disconnected apps with incompatible vocabularies.
Shared primitives are one of the main ways to prevent that.
