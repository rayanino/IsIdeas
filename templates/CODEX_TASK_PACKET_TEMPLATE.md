# Codex Task Packet Template

Use this template when delegating bounded repo work to Codex.

The packet should be precise enough that Codex can execute well without inventing standards.

## Title
A short imperative task title.

## Objective
What exactly should Codex accomplish?

## Why This Matters
Why is this task worth doing in the workshop?

## Repository Context
Which files, directories, or workshop rules matter?

## Inputs To Read First
List the files Codex must inspect before changing anything.

## Required Constraints
List the non-negotiables.
Examples:
- do not change idea stages unless explicitly instructed
- do not invent new ideas
- do not collapse dossier nuance into shallow summaries
- preserve existing IDs and slugs

## Expected Output
What should Codex produce?
Examples:
- a PR
- a consistency report
- updated files
- a drift audit

## Acceptance Criteria
How will the result be judged acceptable?

## Failure Modes To Avoid
What common mistakes would make the task poor?

## Notes For Review
Anything ChatGPT or the human reviewer should inspect carefully after completion.
