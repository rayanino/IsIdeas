# Idea Specification Template

Use this template only when an idea has matured enough to deserve formal specification.

## 1. Thesis
One sentence: what the system is and why it matters.

## 2. User Problem
What exact recurring study bottleneck does this solve?

## 3. Why This Matters for Islamic Study
Why is this strategically important in this domain specifically?

## 4. Strategic Value
How does this increase long-term scholarly capacity?

## 5. Scope Boundary
What is in scope and what is explicitly out of scope?

## 6. Primary Users
Who is this designed for?

## 7. Inputs
What data does the system consume?

## 8. Outputs
What does the system produce?

## 9. Core Workflow
Describe the end-to-end user flow.

## 10. Core Entities / Data Model
List the main entities and relationships.

## 11. System Behavior
Describe the main logic: ranking, scheduling, retrieval, matching, comparison, scoring, or other behavior.

## 12. Source-Backed vs Computed Data
What must be directly traceable, and what may be computed?

## 13. Correctness Constraints
What must the system never do incorrectly or silently?

## 14. Human Review Requirements
Where must a human approve, interpret, or override?

## 15. Failure Modes
How can this system mislead, overreach, or waste time?

## 16. MVP
What is the smallest version that proves the core value?

## 17. Later Expansion
What becomes possible after the MVP?

## 18. Evaluation Criteria
How will we know the system is genuinely useful?

## 19. Dependencies
What shared primitives or upstream systems does this rely on?

## 20. Open Questions
What still needs careful design work?

## 21. Claude Code Implementation Brief
Provide a concise build brief for implementation.
