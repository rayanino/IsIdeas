# Focus Discipline

This document limits how much can be active at once.

Without limits, the repository will accumulate motion without depth.

## Primary Focus Rule
At any given time, there should usually be **one primary deep-focus idea**.

This is the idea receiving the most serious current thinking.
It should be visible in `catalog/ACTIVE_FOCUS.md`.

## Active Attention Limit
There should usually be **no more than three idea workspaces** under meaningful ongoing deep attention.

This does not limit:
- quick captures,
- parked ideas,
- or already stable handoff-ready material.

It limits the number of ideas actively consuming serious architectural attention at once.

## Why This Exists
The limiting factor is not storage.
It is disciplined attention.

Too many active fronts create:
- shallow dossiers,
- fuzzy state tracking,
- false progress,
- and weaker judgment.

## Exceptions
It can be reasonable to exceed the limit briefly when:
- performing repo-level audits,
- comparing closely related ideas,
- or reorganizing the portfolio.

But the normal steady-state should remain narrow.

## Operational Rule
If a new idea wants to enter deep attention beyond the limit, one of these should happen first:
- another active idea is parked,
- another active idea is promoted,
- or another active idea is explicitly deprioritized.
