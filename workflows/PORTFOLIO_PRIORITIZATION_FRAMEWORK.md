# Portfolio Prioritization Framework

Not every strong idea should be pursued at the same time.
This framework exists to rank ideas by strategic timing, not just by abstract appeal.

## Core Principle

Priority is a function of both **strength** and **timing**.
A very strong idea may still be the wrong current focus if it depends on missing substrate or if another idea unlocks more downstream value first.

## Priority Dimensions

### 1. Leverage
If this worked well, how much would it matter over years of study?

### 2. Timing
Is now the right time for this idea, given the current state of `kr`, the rest of the portfolio, and available attention?

### 3. Dependency Position
Does this idea unlock others, or does it depend heavily on things not yet mature?

### 4. Correctness Value
Does it improve correctness, traceability, disciplined learning, or protection from error?

### 5. Implementation Tractability
Can it be matured and eventually built without requiring too much speculative foundation first?

### 6. Risk Profile
How likely is it to create false confidence, overreach, or wasted complexity?

## Suggested Priority Bands

### P1 — Current strategic focus
High leverage, good timing, and a good use of current attention.

### P2 — Strong, but not now
A serious candidate that should stay alive but not dominate present focus.

### P3 — Worth preserving
Potentially valuable, but not currently near the front of the line.

### Parked
Not active now, preserved with explicit reason.

### Rejected
Not worth further serious investment under current understanding.

## Rule
A lower-priority idea is not necessarily a weaker idea.
It may simply be less timely.
