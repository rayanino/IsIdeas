# Workspace Contract

Each serious idea workspace should remain small, disciplined, and legible.

This document defines the minimum contract for a serious idea workspace.

## Required Files

Every live serious idea workspace should contain:
- `README.md`
- `DOSSIER.md`

## Conditionally Allowed Files

Create these only when they are deserved:
- `SPEC.md` — only when the idea reaches formal specification work
- `RESEARCH.md` — when external reading or comparison materially matters
- `SESSIONS/` — when preserving multi-session history adds real value

## Workspace README Contract

The workspace `README.md` should state, at minimum:
- idea ID
- human-readable idea name
- current stage
- current priority band
- why the idea exists
- current status summary
- main risk right now
- next move

## Dossier Contract

The dossier should contain, at minimum:
- working thesis
- why the idea might matter
- the bottleneck
- strongest reasons it may be strong
- strongest reasons it may be weak
- main risks
- relationship to `kr`
- current open questions
- current judgment
- next move

## Sync Rule

The workspace README should stay aligned with the live registry on:
- stage
- priority band
- main risk
- next move

## Minimalism Rule

A workspace should not accumulate files merely to look sophisticated.
Every added artifact should either:
- sharpen judgment,
- preserve important history,
- or enable implementation readiness.
